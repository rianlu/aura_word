# 灵听单词 (AuraWord) - 项目指南 (v2.2)

本文档为开发人员和 AI 代理提供 **灵听单词 (AuraWord)** 项目的核心上下文及实战逻辑。

## 项目概览

**灵听单词 (AuraWord)** 是一个垂直于初中同步英语市场的智能练习工具。其核心商业逻辑是：针对教材换新带来的难度挑战，通过“听说写”三模闭环和 SM-2 算法，为特定地区（如县城周边）的学生提供精准、高效的提分训练。

### 核心竞争力
- **垂直适配**：深度同步 2024/2025 版外研社（FLTRP）教材单词数据。
- **硬核闭环**：强制用户在复习时完成“选义-口语-拼写”三个环节，而非简单的卡片翻看。
- **痛点打击**：通过自动化的听说练习和未来的“听写”功能，降低家长辅导门槛。

---

## 技术架构

- **框架**：Flutter (Dart)
- **数据库**：SQLite (sqflite) 本地化存储，支持极速离线体验。
- **音频逻辑**：
    - 策略：`Youdao API` (优先) -> `Google Oxford API` -> `System TTS` (兜底)。
    - 缓存：所有网络音频均支持本地缓存。
- **安全性**：AES 加密 + HMAC-SHA256 完整性校验，用于 `.wcc` 备份文件。

---

## 目录结构

- `lib/core/`：全局单例服务（音频、备份、统计等）及 UI 主题（极简白底风格）。
- `lib/features/`：功能模块。
    - `practice/`：实现“三模闭环”练习的核心逻辑。
    - `home/`：同步教材任务管理中心。
- `assets/data/`：基于 `books_manifest.json` 的新教材 JSON 数据库。

## 开发与推广规范

### UI 风格指南
- **背景**：保持大面积留白 (`AppColors.background`)。
- **交互**：使用高对比度的 Primary Blue (`0xFF257bf4`) 和 Secondary Yellow (`0xFFffcc00`) 作为操作引导。

### 动效与交互系统 (Design & Motion System)
为了维持项目“灵动、Q弹”的品牌感，所有交互需遵循以下原则：
1. **触觉反馈 (Bouncy Feel)**：所有核心按钮（如 `BubblyButton`）必须具备缩放动画（典型值为 `scale: 0.95`）。
2. **节奏律动 (Rhythmic Motion)**：加载、切换等过渡状态应参考 `SplashScreen` 的“呼吸感”，避免硬切。
3. **物理属性 (Physics-based)**：优先使用 `Curves.easeOutBack` 或 `Curves.elasticOut` 以获得自然的弹簧效果。
4. **层级悬浮 (Floating Cards)**：重要信息承载卡片应具备微小的位移动画（如 `slideY` 或漂浮效果）。

### 数据维护
- 教练模式已废弃，统一使用 **“练习站/训练流”** 作为产品术语。
- 随着教材迭代，需及时更新 `assets/data/` 下的 JSON 文件。
